Servis sağlayıcıları, IP'leri DNS veritabanlarından Client'lere gönderirler. Örneğin; Superonline, Google vs. kendi DNS veritabanlarına sahiplerdir. DNS Client, DNS Server'dan hizmet alan cihaz anlamına gelir.

Router'ları DNS Client yapmak

Router
> en
> conf t
> ip DNS_Server 192.168.1.10

Client'lerde DNS kullanmak çok gereklidir. Router'larda da gereklidir ancak şart değildir. Çok sayıda cihaz var ise DHCP ile otomatik IP dağıtarak işimizi kolaylaştırabiliriz. DHCP Server IP konfigürasyonunu otomatik yapar.

Router'ı DHCP Server'a çevirmek

Router
> ip dhcp pool Pool-Kadikoy
> network 192.168.1.0 255.255.255.0
> default-router 192.168.1.1
> dns-server 192.168.1.10
> exit
> ip dhcp excluded-address 192.168.1.1 192.168.1.10

Hangi Network'e ait IP'leri dağıtmamız gerektiğini gireriz. Default Router ise Gateway bilgisidir. Ardından DNS Server'ın IP'sini gireriz. En son olarak Pool'a dahil etmek istemediğimiz IP aralığını gireriz.

Client'leri IP konfigürasyonu DHCP'ye ayarlıyoruz. DNS hizmeti veren makinelerin IP'leri sabittir bu sebeple Static IP verilir.

Client'ten aldığı IP konfigürasyonu sorgulamak için;

> ipconfig
> ipconfig /all -> komutları kullanılır.
> ipconfig /renew -> Client'in IP almasını sağlamak için kullanılır.

"ping" komutu, ICMP protokolü kullanır.

#debug ip icmp -> açar
#no debug all -> kapatır

BU NOKTADA SUBNETTINGIN DETAYLARINI YAZMAK GEREKIYOR

D Class IP'ler sadece Multicast yayınlar için kullanılır.
D Class: 224.0.0.0 - 239.255.255.255
E Class geliştirici kurullarının çalışmaları için ayrılmıştır.
240.0.0.0 - 255.255.255.255
A, B, C Class'lardan IP verebiliriz. Bunlara Classfull IP adıverilir.
A) 0.0.0.0 - 127.255.255.255/8-16-24
B) 128.0.0.0 - 191.255.255.255
C) 192.0.0.0 - 223.255.255.255
D) 224.0.0.0 - 239.255.255.255
E) 240.0.0.0 - 255.255.255.255
127 ile başlayan IP'ler kullanılamaz. 127 Loopback Address'tir. 127'li IP'lerin tamamı Loopback'tir. 127.1.2.3'te, 127'li bir IP olduğu için Loopback yapabilir. 127, Layer'dan geçemez direkt Ethernet Port'unu Ping'ler. Cevap gelmezse Ethernet Port'u arızalı olabilir. 127'li IP'lerin tamamının Loopback olması 2^24 kadar IP'nin israf olmasına sebep oluyor. Dolayısıyla aslında teknik olarak bir hatadır.

Modemin dış bacağına A, B, C Class'tan gerçek bir IP atamanmak zorundadır. Çünkü internet bağlantısı için birbirinden farklı IP'lere ihtiyaç vardır. Bu IP'leri kiralamanın da belli bir maddi bedeli vardır. İnternete çıkamayan Private IP blokları da vardır. Network tasarlanırken tüm cihazlar için Private IP'ler kullanılır. Bunlara iç IP'ler denir ve şu aralıkta olurlar;
10.0.0.0 - 10.255.255.255
172.16.0.0 - 172.31.255.255
192.168.0.0 - 192.168.255.255
Bu aralıktaki hiç bir IP direkt internete çıkamaz.
ISP bu IP'leri ayırmıştır. Ancak bu içeriye ücretli IP yani Real IP verilmez. Sadece modemin dışarıya açılan bacağında gerçek bir IP'ye ihtiyaç vardır. Şirketin içinde kendi kurduğumuz Network'te bu ücretsiz IP'lerle konfigürasyon yaparız. Bu makinelerin hepsi sadece tek bir gerçek IP ile internete çıkabilirler. ISP size atadığı Static IP'yi başkasına veremez fakat hizmet sağlayıcı sizin modeminize yine gerçek IP vermek zorunda değil. Çünkü evimizdeki modemden örneğin Türk Telekom'un HOP'una kadar aslında Local Network gibidir. türk Telekom'un dış bacaklarında gerçek IP olmak zorunda. Türk Telekom aslında ona bağlı ağlardaki tüm modeleri gerçek IP üzerinden internete çıkartır.
