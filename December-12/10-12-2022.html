NAS ve SAN hakkında Bilgisayar Kavramlarını izleyerek not al!!

NAS(Network Attach Storage)

Ortak Storage'a Network'ten erişilebilir. Bu tarz Storage'lara NAS denir.

SAN(Storage Area Network)

Fiziksel makineler, fiziksel Storage'a fiber üzerinden erişebilirler.

Makineler üzerinde NAS kullanımı

3 adımdan oluşur.
1.adım donanımı satın almaktır.
Örneğin üzerinde 3 tane 1'er Terabyte'lık disk takılı makinemiz var ve bu makineyi Storage olarak kullanırsak bu makineye artık NAS adıverilir.
Fiziksel olarak hazır NAS'lar da vardır. Bu donanım professiyonel ortamlarda kullanılır.
Bir Server da NAS olabilir(Bu şekilde kullanım Software NAS'tır). Network'ten erişilirse NAS, direkt fiberden erişilirse SAN olarak isimlendirilir.
ISCSI Fiziksel kablo üzerinden makineye bağlanma teknolojisidir.
2.adım; ISCSI Targer Konfigürasyon'udur(Hangi makinenin ne kadar disk kullanabileceğini belirler
3.adım; izin verdiğimiz Server'ın NAS ya da SAN'daki ona ayrılmış kapasiteye bağlanma aşamasıdır. Bu işleme ISCSI Initiator denir.
Target işlemi Storage'ta yapılır.
ISCSI Initiator ise Server'da(Local) yapılır.

NAS Server'da yapılacaklar

Server Manager > File and Storage > ISCSI -> ISCSI Default'ta kurulu değildir.
Add Feature > Server Roles > File and Storage > File and iSCSI Services -> iSCSI Target Server, iSCSI Target Storage Provider -> NAS Server'a yüklüyoruz.

iSCSI Wizard'ı çalıştır -> Disk görünmezse; Rescan yap -> Disk sekmesi
Örneğin G diskini kullanacağız.
Bu noktada G sürücüsünün altında iSCSI Virtual Disk adında bir Folder oluşturur(NAS1), o Folder'ın içerisine kurulacak VHD oluşturur.
Bu diskin ne kadarını kullanabileceğimizi belirliyoruz.
iSCSI Target belirliyoruz.(Bu alanı kim belirleyecek?)
Target Name(Target_DC)
Access Server ekliyoruz > Add > Add Initiator ID > Enter a value... > IP Address -> Buraya Target'ın IP Adresi yazılır.(10.3.34.103)
Bu aşamada Target Server'ın açık olması zorunlu değildir.
Bu IP'ye özet IQN adında Unique bir numara verir.
NAS ile Target arasında Target'ı tarif eden benzersiz bir numaradır.
IQN ile Target'ı tarif edebiliriz.
Authentication > CHAP(Target'ın NAS'a giderkenki Authantication'ı)
OK

Target'ın NAS'a bağlanabilmesi için yapılacaklar

Bu makineye iSCSI Service'i kurulmak zorunda değildir.
iSCSI Initiator'e git > Target: 10.3.34.123
Volume and Devices > Auto Configure -> Bağlantı kurulur.

File Server

Bir makineyi File Server yapar.
İdeal olan yani doğrusu Server OS kullanmaktır fakat Client OS'ları da File Server yapılabilir.
Bir Server, Service verebilmesi için Server'a o Service kurulmalıdır. File Service kurulu olarak gelir -> Çok kullanıldığı için yüklü gelir.
(File and Storage Services)
Eğer silindiyse tekrar kurana kadar çalışmaz. Oluşturduğumuz Storage'ın üzerine Folder oluşturduk. O klasörün içerisine File oluuşturduk. Network'te tek bir File paylaşılamaz, ancak bir Folder paylaştırılabilir. Bir Folder paylaştığımız anda paylaşımı yapan makine Network'teeki diğer makineler için File Server olur.
Folder > Right Click > Advanced Sharing
Klasörler Default'ta Share'a kapalıdır.
Default'ta Share Name Folder'ın ismi ile aynıdır. Share Name aynı olmak zorunda değildir. Eş zamanlı erişebilecek kullanıcı sayısı belirlenir. Bu Client OS'lerde 10, Server'larda 16000 civarındadır.

User, File Server'a erişecek makine

IP ya da isim yazılır. -> Gitmek istediğimiz File Server'ın -> 10.3.34.100 -> Paylaşımları listeler.
Folder'ı oluşturan Owner'dır.
Tam yetkilidir.
İstediği kullanıcıyı yetkilendirebilir.
Çok kişiye defalarca göndermek kyerine paylaşım yapılır.
